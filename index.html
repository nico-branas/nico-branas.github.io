Résumé du projet

Run Chicken est un projet robotique dont le but est de développer un canon à eau repousseur de poules. L'objectif est d'empêcher les poules — ou d'autres espèces d'animaux — d'accéder à certains endroits, comme une terrasse, en les dissuadant grâce à un jet d'eau automatique.

Le projet est actuellement en phase de développement sur simulateur : ROS2 Jazzy et Gazebo Harmonic. (Il est donc codé en C++ et Python)

Etapes de développement à venir

  • Intégration sur Raspberry Pi
  • Conception du prototype sur Solidworks
  • Impression 3D du châssis et des pièces mécaniques
  • Réalisation d'un prototype physique

Avancée du projet

1. Création du robot dans Gazebo

J'ai créé une simulation de ce futur robot sous la forme d'un fichier .urdf comprenant deux axes de rotation et une caméra. Ce modèle est ensuite placé dans un monde .sdf Gazebo pour tester ses mouvements dans un environnement virtuel.


2. Architecture du projet sur ROS2

Sur ce graphique on peut constater qu'aujourd'hui l'architecture du projet est divisée en deux sous systèmes principaux:

  • /perception qui analyse les images reçues, détecte les cibles et propose des trajectoires à suivre.
  • /control qui commande le robot en fonction des données reçu par le clavier ou par /perception.

Ros2 contrôle également un élément de la simulation /horse, qui va permettre de tester le suivi d'objet.

La simulation Gazebo communique avec ROS2 via un bridge bidirectionnel (/ros_gz_bridge): les données capteurs (caméra et commandes clavier) sont publiées sur des topics Gazebo convertis en topics ROS2, tandis qu'inversement, Gazebo reçoit les commandes en s'abonnant aux topics correspondants.


3. Détection d'animaux avec YOLOv8 et OpenCV

J'ai intégré le modèle d'intelligence artificielle YOLOv8 pour reconnaître les animaux en temps réel et récupérer des données utiles (position sur l'image, boîte englobante…) grâce à OpenCV. Il existe des versions plus récentes de YOLO, mais le projet ayant pour vocation d'être importé sur Raspberry Pi, j'ai préféré éviter les modèles les plus lourds.


4. Intégration du code IA dans ROS2 / Gazebo

Ce code de détection a été intégré dans un nœud ROS2 afin de le tester directement dans Gazebo. Pour ce test, un cheval a été utilisé à la place d'une poule (un modèle de cheval étant disponible dans Gazebo Fuel), afin de vérifier que le robot est bien capable de suivre un animal en mouvement.
On observe une légère latence de la détection d'image, mais elle reste acceptable pour le suivi en temps réel. Par contre, la latence qu'on observe dans la simulation est due à la capture vidéo.

5. Suivi de trajectoire

Ici, le robot est capable de suivre un animal via la caméra pour le garder en ligne de mire du canon à eau. L'asservissement est aujourd'hui simplement proportionnel, ce qui est largement suffisant pour notre besoin. Je pourrais ajouter une composante intégrale et dérivée pour rendre le suivi de trajectoire plus robuste malgré tout, mais aujourd'hui ce qui le limite sa précision n'est pas l'algorithme de suivi de trajectoire, mais une latence d'environ 140ms entre la capture de l'image et l'arrivée à la phase de traitement et d'une deuxième latence de 180ms liée à la reconnaissance d'image relativement gourmande en temps de calcul.
